# Real Estate Price Prediction with MCP and LLM

A complete demonstration of a property price prediction system (based on a dataset of [Egypt Real Estate Market](https://www.kaggle.com/datasets/waddahali/real-estate-listings)) that combines a trained scikit-learn pipeline, a FastMCP server, and a choice of large language models (local via Ollama or cloud via OpenAI/Anthropic). Users interact through a web chat interface.

## Project Structure

```plaintext
.
├── regression_model_VF.ipynb # The notebook detailing the process to create the pipeline.
├── pipeline.py # Custom preprocessing pipeline class (must be imported for app.py to work)
├── server.py # FastMCP server exposing the prediction tool
├── app.py # FastAPI web application (chat + LLM orchestration)
├── price_prediction_model.joblib # Trained pipeline (generated by regression_model_VF.ipynb)
├── .env # Environment variables (API keys, model names)
├── requirements.txt # Install dependencies
└── README.md

```

## Features

- **Local ML model** : Predicts property prices using a scikit-learn pipeline.
- **MCP integration** : Exposes the prediction as an MCP tool (`estimate_price`).
- **Multiple LLM backends** : Choose between:
  - Ollama (local, default model : llama3.2:3b)
  - OpenAI (default model : GPT-4o-mini)
  - Anthropic Claude (default model : Claude 3 Haiku)
- **Web chat interface** : Clean chat UI with provider selector and API key input.
- **Default values** : Missing parameters are filled with sensible defaults.
- **Fully offline capable** : Works without internet when using Ollama.

## How It Works

1. User asks a question in the web UI (e.g., “What is the price of a 80m² apartment with 2 bedrooms in Cairo ?”).
2. The selected LLM analyses the question and decides to call the MCP tool `estimate_price` with extracted parameters.
3. `app.py` forwards the call to the MCP server (`server.py`), which runs the trained pipeline.
4. The predicted price is returned to the LLM, which reformulates it into a natural answer.

## Prerequisites

- Python 3.10+ with `conda` or `venv`
- [Ollama](https://ollama.com) (optional, for local LLM)
- API keys for OpenAI / Anthropic (optional)

## Setup

### 1. Clone the repository
```bash
git clone https://github.com/Nicolas-Perion/Real-Estate-Price-Prediction-with-MCP-and-LLM
cd real-estate-mcp
```

### 2. Create a python environment and install dependencies
```bash
python -m venv mon_env
pip install -r requirements.txt
```

### 3. Configure environment variables
Set the values of the models in the ```.env``` file depending on what LLM you want to use.

### 4. (Optional) Pull a local Ollama model (if you want to use and run the system locally)
In a terminal, run :
```bash
ollama pull <your ollama model>
```

(I recommand phi3.5 or gemma2 for small configurations and Llama 3.1 8B for more powerful ones.)

```bash
ollama pull pparikh2/phi3.5Q4_K_M:latest
ollama pull gemma2:2b
ollama pull llama3.1:8b-instruct-q4_K_M
```

## Running the Demo
Open three terminals (or two if you won't use a local Ollama model) :
| Terminal | Command | Description |
|:---:|:---:|:---:|
| 1 | ```python server.py``` | Starts the MCP server on ```http://127.0.0.1:8000``` |
| 2 | ```python app.py``` | Starts the web UI on ```http://127.0.0.1:8001``` |
| 3 (Optional) | ```ollama serve``` | Starts Ollama's API server to allows other apps to communicate with Ollama's LLMs|

Then open your browser at ```http://localhost:8001``` and chat.

## Testing
Try these example questions:

- “Estimate price for a 120m² villa with 3 bedrooms in Giza”

Important Considerations:

The price prediction model was trained on a dataset from the [Egyptian real estate market](https://www.kaggle.com/datasets/waddahali/real-estate-listings). Therefore, when asking questions, the only available cities are:

- Cairo
- Giza
- North Coast
- Red Sea
- Suez
- Alexandria
- Sharqia
- South Sinai
- Qalyubia
- Matrouh
- Al Daqahlya
- Demyat

Additionally, a size limit for the property area has been set to 5,000 square meters.

## Customisation
- Change the default model : Edit the ```.env``` file.
- Adjust default values : Modify the ```DEFAULTS``` dictionary (dictionary used to set arguments expected by the prediction model to default values if the user ommits them in its question) in ```server.py```.
- Add more LLM providers – Extend ```app.py``` accordingly.
